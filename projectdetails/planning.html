Development Plan

Relational Schema

Genres(GenreID key, Name)

Movies(movieID key, title, year, is_tv, genreID)

Ratings(RatingID key, Rating, Source, MovieID foreign key, numVotes)

Actors(actorID key, name)

AppearsIn(movieID key, actorID key, role)

Following(Follower key, Following key)

UserLikes(user_id key, movie_id key)

Users(id key, username, password)

Databases/Software Platforms

We will be using the default tools on dsg1 for the database of our project. We are planning to use python and flask for the front-end framework.

Data Location

We are planning to retrieve our data from the following sources:

  • IMBD hosts their data on AWS.
  • User data from those using the site

Labor Division

We plan to work together on most everything to ensure equal labor. We will put two team members primarily in charge of the database and querying and two team members in charge of the front-end web application. They all four members will work together on the two advanced functions.
